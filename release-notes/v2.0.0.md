# TankAlarm v2.0.0

**Theme: dashboard data integrity — trustworthy voltage and sensor reporting.**

This release fixes a set of issues that let the operator dashboard display stale or fabricated
values (a stuck `VIN 4.69V`, a false `0.0 PSI`, and a `sensor-fault` whose cause was unclear). It
also hardens both the client and server so a bad or out-of-date reading can no longer linger on the
dashboard. It builds on the OTA reliability work delivered across v1.9.30–v1.9.42.

Driven by the analysis in `CODE REVIEW/CODE_REVIEW_06222026_VOLTAGE_ALARM.md`; deferred hardware and
configuration items are tracked in `CODE REVIEW/CODE_REVIEW_06222026_DEFERRED_ITEMS.md`.

---

## Highlights
- **The dashboard no longer shows a stale ~5 V "battery" voltage.** The Notecard 5 V rail can never
  be reported as a battery reading, and the server ages out / plausibility-checks voltage before
  display.
- **Unpowered analog sensors now fault instead of reporting a fake `0.0`.**
- **Faulted or reused sensor readings are flagged** so the server never treats them as authoritative.
- **Diagnostic alarms (`sensor-fault` / `sensor-stuck`) are no longer wrongly cleared** by a daily
  high/low report.

---

## Client changes (`TankAlarm-112025-Client-BluesOpta`)
- **Battery voltage source hardening.** `getEffectiveBatteryVoltage()` now compile-gates the Notecard
  `card.voltage` source out on Opta/mbed, so the regulated ~5 V carrier rail can never be treated as
  a battery reading. Battery voltage comes only from the SunSaver MPPT (RS‑485) or the analog Vin
  divider.
- **Analog live-zero guard.** `readAnalogSensor()` returns `NAN` (→ `sensor-fault`) when an
  elevated‑minimum transducer (e.g. a 1–5 V sensor) reads far below its minimum, instead of mapping
  an unpowered 0 V to a clamped, fabricated `0.0`.
- **Voltage source tagging.** Telemetry and daily reports now tag the voltage source
  (`vs = "mppt" | "vin-divider"`) whenever a voltage is sent.
- **Data-quality flags.** Sensor notes now include `sf` (sensor in a failed state) and `ru` (reading
  reused from a previous cycle) so the server/dashboard can distinguish a fresh valid reading from a
  stale one.

## Server changes (`TankAlarm-112025-Server-BluesOpta`)
- **Voltage display gate.** The dashboard API only exposes a client's voltage when it is **plausible
  (≥ 6.0 V)** and **recent (< 48 h old)**. A value the client has stopped sending now ages off the
  dashboard on its own, and one implausible reading can no longer pin the display indefinitely.
- **Voltage ingest gate.** Incoming voltage is stored only when it is source‑tagged (`mppt` /
  `vin-divider`) or plausibly a battery (≥ 6.0 V); the MPPT pack voltage (`solar.bv`) is always
  trusted. This prevents a misbuilt/old client from re‑poisoning the dashboard.
- **Stale raw‑mA cleanup.** A current‑loop daily report that omits raw `ma` now clears the stored
  value, so a pegged/stale milliamp reading cannot linger.
- **Faulted level protection.** A current‑loop daily `lvl` flagged faulted/reused with no valid `ma`
  no longer overwrites the last good level (or records a misleading history snapshot).
- **Diagnostic alarm safety.** The daily high/low alarm reconciliation no longer clears
  `sensor-fault` / `sensor-stuck` alarms — those are not represented in the high/low array and were
  being cleared incorrectly.

## Known issues / deferred (not in this release)
- **Recurring `sensor-fault` on the A0602 current‑loop module is a hardware/I2C‑bus issue**, not a
  firmware logic bug. v2.0.0 makes the dashboard report it *accurately*; the underlying bus
  reliability needs a bench investigation. See `CODE_REVIEW_06222026_DEFERRED_ITEMS.md` (Fix 11) for
  the checklist and suggested diagnostics.
- Config‑generator battery default and an optional one‑shot "clear voltage" admin action are tracked
  in the same deferred‑items review (Fixes 4 and 5/15). The stale value now self‑heals via the serve
  gate, so the manual clear is optional.

---

## Upgrade & deployment notes
- **Server:** flash over USB (COM3). After flashing, any previously stuck `4.69 V` / `18.02 mA`
  values age off the dashboard within 48 h (or instantly on a server reboot).
- **Client:** flash the signed `TankAlarm-Client-secure-v2.0.0.slot.bin` via OTA, or
  `TankAlarm-Client-v2.0.0.bin` over USB. Client config is preserved across the update (stored on a
  separate QSPI partition).
- **Compatibility:** the new note fields (`vs`, `sf`, `ru`) are additive and backward‑compatible —
  an older server simply ignores them, and the v2.0.0 server gates on their presence.
