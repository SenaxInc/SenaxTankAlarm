# TankAlarm v2.0.42

**Theme: make the SunSaver MPPT (RS‑485) link observable from Notehub.**

This release adds diagnostics so an enabled‑but‑not‑communicating Modbus MPPT charger is visible
remotely. The firmware logic for reading the SunSaver is unchanged; what's new is that the client now
reports the Modbus link health instead of failing silently.

---

## The problem

When the SunSaver MPPT is configured (`solarCharger.enabled = true`) but the RS‑485/Modbus link does
not respond, the client produced **no observable signal at all**:

- `getEffectiveBatteryVoltage()` returns 0, so no voltage (`v`) is sent.
- `appendSolarDataToDaily()` omitted the entire `solar` block when communication failed.
- The comm‑failure alarm (`alertOnCommFailure`) defaults to off.

The result: an enabled‑but‑failing charger looked **identical to a disabled one** on the dashboard
and in the event log — impossible to diagnose without a site visit to a field client.

## The fix (client diagnostics)

- **Telemetry** now includes `scOk` whenever the charger is enabled: `scOk:1` = Modbus link healthy,
  `scOk:0` = enabled but not communicating. This appears on every telemetry cycle, so the link state
  is visible within one sample interval.
- **Daily report** now emits a minimal `solar` status on failure (`{"commOk":0,"errs":N}`) instead of
  omitting the block, and tags the healthy block with `"commOk":1`.

No change to the RS‑485 driver, register map, scaling, or timing — this is purely additive
observability so the next telemetry/report tells us whether the charger is enabled, communicating,
and (if not) how many consecutive errors it has seen.

## Deployment / how to read it
After CI builds the release, assign `TankAlarm-Client-secure-v2.0.42.slot.bin` on Notehub to the
client. Then in the telemetry notes:

- **No `scOk` field** → the charger is **not enabled** in the client's applied config (re-push a
  "Solar + Modbus MPPT (RS‑485)" config).
- **`scOk:0`** → enabled but the SunSaver isn't answering → check RS‑485 wiring (A/B), the MRC‑1
  power, slave ID/baud, or the driver timing.
- **`scOk:1`** (and `v` present) → the link is healthy and voltage is flowing.
