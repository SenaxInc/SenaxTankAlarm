# TankAlarm v2.0.44

**Theme: read the SunSaver MPPT one register at a time — match the bench‑proven transaction.**

This release changes how the client reads the SunSaver MPPT over RS‑485/Modbus so it matches the
exact transaction that was verified on the bench in April 2026. It targets the field symptom where
the charger is enabled and the poller runs, but every poll reports `scOk:0` (no communication).

---

## The discrepancy this fixes

A comparison of the **working bench test code** against the **production driver** found one
functional difference:

- The bench tools (`firmware/sunsaver-modbus-test`, `firmware/sunsaver-rs485-raw`) — the code that
  *successfully* read the SunSaver via the Morningstar MRC‑1 on 2026‑04‑22 — read registers **one at
  a time** (`requestFrom(..., address, 1)`).
- The production driver read the five real‑time registers (`0x0008–0x000C`) as a **single
  multi‑register block** (`requestFrom(..., 0x0008, 5)`).

A multi‑register block read is a different Modbus transaction, and the **MRC‑1 MeterBus→RS‑485
bridge does not reliably answer it**. Tellingly, production's own *startup probe* uses a single
register read (which succeeds) while every *poll* used the block read (which fails) — exactly
matching the field signature: config applied, poller enabled, but `scOk:0` on every sample.

## The fix

- New helper `readRegistersIndividually()` reads N contiguous registers as **N single‑register
  Modbus transactions** (the bench‑verified shape). It returns success only if every register reads,
  **short‑circuits on the first failure** (so a dead bus still costs only one read, not five), and
  **kicks the watchdog before each read**.
- `readRegisters()` now uses it for both the real‑time block (`0x0008–0x000C`) and the one‑time
  charge‑setpoint read. The v2.0.43 noise‑immunity retry and the FC03↔FC04 fallback are preserved.

No change to baud/parity (9600 8N2), `setDelays(0, 1200)`, slave ID, register map, or scaling — those
already matched the bench. The watchdog timing is unchanged in the worst case (a dead bus still fails
on the first register and short‑circuits).

## How to verify after OTA
Assign `TankAlarm-Client-secure-v2.0.44.slot.bin` to the client. Then in telemetry:
- **`scOk:1`** (and a `v` voltage value) → the single‑register reads work → **fixed**.
- **`scOk:0`** still → the link problem is elsewhere (wiring/MRC‑1/SunSaver), and the next step is a
  bench session with the `sunsaver-rs485-raw` workhorse.
