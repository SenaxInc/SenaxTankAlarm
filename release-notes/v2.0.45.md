# v2.0.45 — Field diagnostics & the SunSaver `scOk:0` regression fix

**Release A of the v24 field-diagnostics plan** (`CODE REVIEW/REMAINING_PROBLEMS_AND_ATTEMPTS_06232026.md`).
Diagnostics + verified‑safe corrections only. **SunSaver RS‑485 transport is unchanged** (`setDelays(0,1200)`,
9600 8N2, slave 1, single‑register reads, FC03/FC04 fallback). No bench hardware required to deploy.

## Headline fix — `scOk:0` on healthy hardware (signed current regression)

`SolarManager::scaleCurrent()` interpreted the SunSaver filtered current registers
(`adc_ic_f` 0x000B / `adc_il_f` 0x000C) as **unsigned**. A small negative charge current near zero
(normal at night / no‑charge, e.g. `0xFFF0` = −16 counts) scaled to `65520 × 79.16 / 32768 ≈ 158 A`, which
tripped the plausibility clamp and silently failed the **entire** poll → `communicationOk = false` → `scOk:0`,
even though the Modbus read succeeded. The April bench test (daylight, positive current, no clamp) never hit it.

- **`scaleCurrent()` now casts through `int16_t`** (two's‑complement), so near‑zero/negative currents scale
  correctly; the realtime block saturates small negatives to 0.
- **Transport‑health gate decoupled from data quality:** only **battery voltage** (5–40 V) decides `scOk`. Array
  voltage and currents are clamped as data‑quality fields, never used to fail the poll.

## SunSaver driver (`TankAlarm_Solar.cpp/.h`)
- **Skip unused realtime `0x000A`** (load voltage was read but never consumed; a NACK on it could fail the poll).
  Realtime reads now exactly mirror the bench‑proven sketch: `0x0008, 0x0009, 0x000B, 0x000C`.
- **Skip setpoint gap `0x0034`** — read `0x0033 / 0x0035 / 0x0036` explicitly (best‑effort; never fails the poll
  nor perturbs the FC cache).
- **Preserve the FC03/FC04 cache on a transient failure** — a single timeout on a known‑good FC no longer forces
  permanent re‑probing of the other function code on every poll.
- **Modbus error taxonomy** — capture `ModbusRTUClient.lastError()`, the failing register address, response time,
  and an error count, plus a CRC‑valid‑but‑rejected (`scImpl`) flag. Exposed via new `SolarManager` accessors and
  emitted in telemetry (`scImpl`) and the daily solar‑failure block (`merr` / `maddr` / `mms` / `merrTxt`). This
  lets a field `scOk:0` be classified **remotely**: `timeout` ⇒ physical link, `illegal data address` ⇒ register/
  slave, `invalid CRC` ⇒ wire noise.
- **Drain the RS‑485 RX buffer** before each Modbus transaction (the library flushes on CRC/FC error but not on
  timeout).

## A0602 current‑loop (`TankAlarm_I2C.h`, client)
- **Confirm the channel is actually in current‑ADC mode** after `SET CH_ADC` via the framed
  `GET_CHANNEL_FUNCTION (0x40)` protocol (the SET ACK only means "queued"). Frame + the `CH_FUNC_CURRENT_INPUT_EXT_POWER = 0x04`
  constant were verified against the installed `Arduino_Opta_Blueprint` source.
- **Graceful, non‑bricking gate:** hard‑fault (NAN) only if the config SET is NACKed every attempt **or** the
  expansion *positively* reports a non‑current‑ADC function. If the A0602 firmware doesn't answer `0x40` at all,
  the read proceeds (the framed `GET ADC` keeps its own CRC/channel guard) so older expansion firmware is never
  bricked.
- **Framed‑path failure accounting** — `gCurrentLoopI2cErrors` now also counts framed‑read failures, so a
  silently‑failing A0602 shows up in `i2c_cl_err` / `i2c-error-rate` instead of looking "reused‑only".
- **P1 power‑OFF retry** (×3, mirroring the existing ON retry) so a transient NACK can't leave the transmitter
  powered.
- **Defer Notecard outbox trimming** until after sampling on current‑loop devices, and drain the Wire RX buffer
  first, to keep the heavy Notecard I²C burst off the timing‑sensitive A0602 window. A0602 transactions use a
  scoped 100 ms Wire timeout, restored before every return.
