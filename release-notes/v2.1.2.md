# v2.1.2 — Hotfix: wrong mA scale on DAC loop-powered reads (10.245mA telemetry bug)

**Client + Common hotfix release.** v2.1.1 transmitted a wrong mA value for
internally-powered (DAC loop) 4-20mA sensors — e.g. a real 4.51mA loop was
reported as **10.245mA**. Externally-powered sensors were unaffected. Flash
the client; server build unchanged apart from the version string.

## Root cause

The A0602 uses **two different ADC scales** depending on the channel mode
(expansion firmware `parse_setup_adc_channel()`, Arduino_Opta_Blueprint):

| channel mode | AD74412R ADC range | raw → mA |
|---|---|---|
| plain current ADC (externally powered) | unipolar 0-2.5V over 100Ω | `25 * raw / 65535` |
| voltage DAC + added current ADC (internally powered loop) | **bipolar ±2.5V** over 100Ω | `(raw/65535)*50 − 25` |

In the internally-powered mode, zero current sits at **mid-scale** (~0x8000)
and current sourced by the channel reads **negative**. The managed library's
`pinCurrent()` handles this with a dedicated branch
(`((code/65535)*5.0 − 2.5)/100` A); the bench DAC utility printed
`abs(pinCurrent())` — which is why it showed the correct 4.51mA.

`tankalarm_readCurrentAdcFramed()` applied the **unipolar** formula to both
modes. For the same raw sample (26857) that the utility converted to 4.51mA,
the client computed `25*26857/65535 = 10.245mA` — the exact value seen in
Notehub telemetry. (Physics check: ~3.5ft of fresh water ≈ 1.5psi on a
0-50psi transmitter ≈ 4.48mA. ✓)

## Fix

- New `tankalarm_readLoopPoweredCurrentAdcFramed()` in `TankAlarm_I2C.h`:
  same framed GET + CRC + channel-echo validation, but converts with the
  bipolar scale and returns the magnitude of the sourced current, so the
  existing 4-20mA live-zero / over-range guards work unchanged.
  Rail values `0x0000`/`0xFFFF` (±25mA rails — bus garbage or the
  expansion's not-yet-applied ADC cache) are rejected as faults.
- `readCurrentLoopSensor()` picks the reader by mode: bipolar for
  `loopPowerEnabled` (DAC loop), unipolar for externally powered.
- Diagnosis aid: the client now prints one line per sampling burst —
  `CL ch0: 4.511 mA (n=5/5, loopPower=1)` — so a mis-scaled or mis-wired
  sensor is visible on the serial console without a Notehub round trip.
