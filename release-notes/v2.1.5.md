## Fixes

### 1. Endless setpoint re-probe on firmware revisions with unverifiable setpoint registers

The charge-setpoint read (used by battery-chemistry verification) was gated on
`!setpointsValid` — intended as "read once per session." But on SunSaver firmware
revisions where the coded setpoint addresses (0x0033/0x0035/0x0036) hold other data,
the plausibility check rejects the values every time, so `setpointsValid` never latches
and the client re-read **all three registers and printed a warning on every single poll,
forever** (~4,300 wasted Modbus transactions/day at the 60 s cadence, bench-observed).

The probe now gives up after 3 consecutive CRC-valid-but-implausible results (the values
are a property of the controller firmware and cannot change at runtime) and prints a
final "giving up until next re-init" line. Transport failures do NOT count toward the
give-up, so a flaky bus cannot permanently disable chemistry verification on controllers
where the addresses are correct. The counter re-arms on `SolarManager::begin()`.

`scSpv:0` in telemetry continues to report the setpoints-never-validated state truthfully.

### 2. MRC-1 power-switch warning in the boot hardware-requirements print

The boot print now warns: "POWER SWITCH MUST BE ON (LED is solid green either way;
switch OFF = RS-485 side inert, no comms despite green LED)" — the exact failure mode
that took the bench link down from ~May through 2026-07-03. (Committed on master in
33e916a; first shipped in this release.)

## Review confirmations (no changes needed)

- **Transport:** 8N2, baud-scaled post-TX delay (1195 µs @ 9600), ≥500 ms timeout, RX
  drain per transaction, single-register reads, FC03/FC04 fallback with cache, fastFail
  on dead bus, bounded retry, watchdog kicks throughout.
- **Scaling:** V = raw·96.667/32768 (bench: raw 0x1183 → 13.22 V, matches MSView);
  currents signed via int16_t cast, small negatives saturated to zero.
- **Data quality:** battery-voltage plausibility gate (5–40 V) decides transport health;
  array/current clamps are data-quality only; `scImpl` distinguishes value-rejection
  from link-down.
- **Alerting:** comm-failure-first with stale-data suppression (m-8); `checkAlerts()`
  runs after every poll attempt, so `alertOnCommFailure` is reachable; software-derived
  charge state (FLOAT/BULK/NIGHT/DISCONNECT) from verified registers only.
- **Voltage to server:** `getEffectiveBatteryVoltage()` strict MPPT > Vin-divider
  priority → telemetry `v` + `vs:"mppt"` on every note → server `vinVoltage` + source
  tag → dashboard; daily report carries `v`/`vs` + full `solar` block (`bv`/`av`/`ic`,
  windowed `merr*` buckets reset after each daily report).
- **Unverified registers** (temps/status/faults/daily-Ah at 0x001B+/0x002B+) remain
  compile-gated; live sweep confirmed those addresses return other data on this
  firmware revision (e.g. 0x002B = 0x1027 — voltage-like, not a 0-8 charge state).
