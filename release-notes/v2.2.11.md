# TankAlarm v2.2.11

Fixes every server-webpage bug from issue #313, verified live against a production server. Full diagnosis in `CODE REVIEW/CODE_REVIEW_07202026_ISSUE_313_SERVER_WEBPAGE_BUGS.md`.

## Server Webpage Fixes (issue #313)

- **Client Console restored** — the `/client-console` page had been completely non-functional since March (commit `502bbcd`): a single stray `}` in the page script made the whole script block fail to parse, so the client dropdown and Active Sites never rendered. One-character fix; the page now lists clients, details, and active sites again.
- **Dashboard trend graph fixed** — "No trend data yet" was caused by a phantom `sensorIndex 0` history record (created by legacy notes without a sensor index) colliding with the real sensor via a falsy JS key (`sensorIndex||1`) and overwriting its readings with an empty series. Telemetry/daily ingest now rejects notes without a valid 1-based sensor index, phantom sensor-0 histories are purged when the hot tier loads from flash, and the dashboard key no longer aliases 0 onto 1. Sparklines draw again.
- **Historical Data page restored** — the page required a `sites` map the API never sent; the resulting exception wiped the whole page to zeros ("0 total sensors", empty Sites & Sensors, blank VIN chart) even though the API carried weeks of readings. The page now derives the site grouping client-side; sensor counts, filters, level trends, alarm frequency, and the VIN voltage chart all render.
- **FTP archive banner fixed + new toggle** — "Enable FTP for full archive" showed even with FTP connected because the history-archive flag defaulted off and had **no UI anywhere** to enable it. History archival now defaults on, a new **Archive history to FTP** toggle lives in the Server Settings FTP section (round-trips through `/api/server-settings`), and the banner distinguishes three states: archive active (with "first sync pending" until the first upload), FTP connected but archive off, and FTP not configured.
- **FTP save UX** — new **Save FTP Settings** button with a "✓ Saved" indicator next to Test Connection, and **Test Connection now saves your edits first** (it previously tested the last-saved config, silently ignoring unsaved changes in the form).
- **Server Event SMS Recipients** — the settings-page SMS list was the *same* list as the Contacts page, so server-ops alerts and field alarms could not be separated. There is now a dedicated `serverSmsRecipients` list used for server events only (power loss/restore, OTA stalled/applied/reverted, stale clients and recovery, test SMS). If the list is empty, server events fall back to the general SMS alert list, so existing installs keep alerting unchanged until the new list is populated.
- **Contacts page redesign** — the contact list's card classes had no CSS at all (the reported "bunched up" layout). Contacts now render as a proper table: **Name | Phone | Email | SMS | Email Alerts | ✏️ Edit | 🗑️ Delete**, with alarm associations shown as chips beneath each row. The previously unlabeled "×" now includes the site name in the chip and carries a tooltip explaining it removes that alarm association.
- **Daily Report Editor** — the `/email-format` page (which already drives the daily email layout) is retitled **Daily Report Editor** and is now linked from the Contacts page's master Daily Report card and the settings page.
- Fleet-summary metadata fix: `hotTierSnapshots` now reports true if *any* history slot has data (previously only checked slot 0).

## Upgrade Notes

- Server-only release: flash the server binary. **No client or viewer firmware change** (version bump only) and no Notehub route changes.
- On servers that previously saved history settings, enable **Archive history to FTP** once in Server Settings (new installs default on).
- Optionally populate **Server Event SMS Recipients** on the settings page to separate server-ops SMS from field alarm SMS; leaving it empty preserves existing behavior.
